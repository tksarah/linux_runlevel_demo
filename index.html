LESSON FILE

起動前の GRUB2 指定と、起動後の init / systemctl 遷移を学ぶ

STEP 0 では GRUB2 の編集モードから rescue.target などを指定して起動します。 STEP 1 以降では、Linux が起動している状態で init または systemctl を実行した時の状態変化を確認します。

STEP 0

GRUB2 の編集モードで target を指定して起動

最初は GRUB メニューが表示されています。e を押して編集モードに入り、 linux 行の末尾に systemd.unit=target名 を手動で入力します。

systemd.unit=rescue.target systemd.unit=emergency.target systemd.unit=multi-user.target systemd.unit=graphical.target
...

まず e を押して編集モードに入ります。

STEP 1

起動後の Linux で init / systemctl コマンドを入力

Linux が起動済みの状態を前提に、init または対応する systemctl コマンドを実行します。

#

入力例: init 5 / systemctl isolate graphical.target

REFERENCE

簡単な解説

Runlevel 0: halt / システム停止

init 0 は停止処理です。サービスを止め、ファイルシステムを安全に処理し、電源断できる状態へ向かいます。

補足: 現在の Linux では systemd target で同じような起動状態を扱う環境が多くあります。 GRUB2 から target を指定すると、起動直後から目的の target に入れます。
